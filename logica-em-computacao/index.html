O que é Lógica de Programação?

São conjuntos de passos pensados e organizados de forma lógica, esses passos tornam-se um algoritmo para resolução de um ou mais problemas que foi escrito para ser lido e processado por sistemas computacionais ou qualquer dispositivo hardware programável.

O que seria Lógica?

Minha definição de Lógica, seria a análise e entendimento de algo abstrato, algo inicialmente desconexo, pegá-lo e transformá-lo em algo claro, conexo e que faça sentido.

O que seria um Algoritmo?

Algoritmo é um conjunto de passos lógicos para resolução de um problema ou para execução de uma tarefa. Exemplo simples de um algoritmo:

Algoritmo para beber água.
  • "PEGAR COPO"
  • "PEGAR RECIPIENTE COM AGUA"
  • "ENXER COPO COM AGUA"
  • "LEVAR COPO ATE A BOCA"
  • "INGERIR AGUA"
    • FIM_ALGORITMO

    Simples, mas fica fácil de entender, esse é um conjunto de passos lógicos para beber água. E esse conjunto de passos é um ALGORITMO

    Aplicando a lógica na computação, trago um exemplo simples:

    "Digite seu CPF:"

    [usuário entra com o dado]

    999.999.999-99

    "Data de nascimento usuário X: 01/01/1999"

    Ou seja, o usuário viu a informação "Digite seu CPF" no display, entrou com o CPF que ele desejava obter a informação, o programa leu esse CPF, ligou o CPF através de dados computacionais pré-estabelecidos pelo sistema programado e entregou a informação ao usuário, ou seja, executou um conjunto de passos para entregar a data de nascimento que esse usuário precisava.

    Então a Lógica Computacional pode ser definida como uma forma de resolução de tarefas através de programações pré-estabelecidas com passos lógicos que transformam dados em informações a serem exibidas e entendidas pelo usuário final.

    E programadores se tornam: Indivíduos com habilidades de Lógica Computacional, capazes de entender os problemas e resolvê-los com passos inicialmente lógicos e posteriormente comunicá-los a máquina ou ao interpretador (browser por exemplo) com as Linguagens Básicas ou Avançadas, fazendo a máquina ou interpretador entender o software programado em uma determinada linguagem, e esse software entenderá o problema proposto através de meta-dados pré-estabelecidos no software e entregará a solução e/ou informação ao usuário.
